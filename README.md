### Для автоматизации сборки и управления зависимостями использовать Gradle так как имеет больше гибких настроек. Использовать библиотеку JUnit для настройки автотестов. Для выстраивания непрерывной интеграции и сокращения времени на выдачу результатов о прохождении тестов использовать Continuous Integration (CI). Использовать Selenium для управления браузерами и работа с элементами HTML.
### Для генерации валидных данных для заполнения формы воспользоваться готовыми генераторами (библиотека Faker для Java) или написать самим эти генераторы данных.
### Необходимо разрешение собственника на проведение автотестирования сайта, в противном случае необходимо предоставить демо версию. Необходимы данные по правилам заполнения формы.
### При выполнении автоматизированного тестирования возможны падения тестов из-за низкой скорости или прерывания связи с сайтом, перебои работы тестируемого сайта. Во время написания автотеста на сайте могут быть внесены изменения, которые могут привести к некорректному прохождению тестов.
### Для автоматизации тестирования необходим специалист со знаниями и навыками автотестирования.
### Время на реализацию автоматического тестирования 18 часов.
